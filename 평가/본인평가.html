작업 내용 분석 및 스스로의 평가

어려웠던 점과 극복한 과정

처음에는 태그가 작동하는 기본 구조나 파일 경로를 연결하는 부분에서 헷갈리는 점이 많았습니다. 특히 다른 폴더에 있는 파일들을 링크로 엮어줄 때 경로를 잘못 지정해 페이지를 찾을 수 없다는 에러가 자주 발생했습니다. 하지만 파일 탐색기 구조를 다시 천천히 들여다보며 상대 경로의 개념을 이해하게 되었고, 마침내 원하는 페이지들이 서로 매끄럽게 연결되는 것을 구현했습니다.

또한, 강의 시간표를 만들 때 셀 병합(colspan, rowspan) 속성이 복잡해서 표가 자꾸 깨지는 현상이 있었습니다. 월요일부터 금요일까지 가로줄과 세로줄의 칸수를 메모장에 미리 적어두고 검토하면서 코드를 고친 끝에 형태를 정상적으로 잡아낼 수 있었습니다.

이번 실습을 통해 확실히 배운 점

  • HTML의 기본 구성 뼈대인 head와 body의 역할을 완전히 구별하여 작성할 수 있게 되었습니다.
  • 목적에 맞춰서 순서가 없는 목록(ul)과 순서가 있는 목록(ol), 그리고 정의형 목록(dl)을 유연하게 골라 쓸 수 있습니다.
  • 새 탭으로 링크를 띄워주는 target="_blank" 속성의 쓰임새와 중요성을 알게 되었습니다.
  • VS Code에서 작업한 결과물을 Git과 GitHub을 사용해 원격 저장소에 안정적으로 커밋하고 업데이트하는 흐름을 익혔습니다.

앞으로 더 보완하고 싶은 부분

현재 단계에서는 디자인 요소를 넣지 말라는 가이드라인에 따라 순수 HTML 태그로만 뼈대를 구성하여 화면이 전반적으로 투박해 보입니다. 앞으로 배울 CSS 과목을 적극적으로 공부해서 정렬 상태를 더 다듬고, 보기 좋은 폰트와 배경색을 적용하여 세련된 디자인의 웹페이지로 발전시켜 나갈 계획입니다.

전체적인 소감

단순히 예제 코드를 받아 적는 공부에서 벗어나 직접 레이아웃을 짜보고 제가 자주 쓰는 링크들과 좋아하는 것들을 채워 넣으며 웹을 만드니 공부가 훨씬 흥미로웠습니다. 개발 공부를 처음 시작하며 막연한 두려움이 있었는데, 브라우저에 제가 쓴 코드대로 화면이 바로 구현되는 것을 보며 재미를 붙이게 된 뜻깊은 작업이었습니다.
